## アニメーション入門

p5.js のアニメーションは `draw()` 関数が**毎フレーム繰り返し呼ばれる**ことで実現されます。上のキャンバスでは、ボールが壁に跳ね返りながら動いています。

## draw() ループの仕組み

```
setup()       ← 最初に1回だけ実行
  ↓
draw()        ← 毎フレーム実行（デフォルト 60fps）
  ↓
draw()
  ↓
draw()
  ↓
  ...         ← ずっと繰り返す
```

p5.js はデフォルトで **毎秒 60 回**（60fps）`draw()` を呼びます。各呼び出しで位置を少しずつ変えることで、動きを表現します。

## コードの構造

```js
export default function (p) {
  let x, y, vx, vy;

  p.setup = () => {
    p.createCanvas(400, 400);
    x = 200;  // 初期位置
    y = 200;
    vx = 3;   // x方向の速度
    vy = 2;   // y方向の速度
  };

  p.draw = () => {
    p.background(30, 30, 50, 40); // 半透明の背景

    x += vx; // 位置の更新
    y += vy;

    // 壁で跳ね返る
    if (x < 0 || x > p.width) vx *= -1;
    if (y < 0 || y > p.height) vy *= -1;

    p.circle(x, y, 40);
  };
}
```

### ポイント

1. **変数をスケッチ関数のスコープで宣言** — `let x, y` は `setup` と `draw` の両方からアクセスできます
2. **`x += vx`** — フレームごとに位置を速度分だけ移動
3. **壁の跳ね返り** — キャンバスの端に達したら速度を反転（`*= -1`）

## 半透明背景のテクニック

```js
p.background(30, 30, 50, 40);
```

`background()` の 4 番目の引数は **透明度** です。不透明（255）にすると前のフレームが完全に消えますが、小さい値にすると**前のフレームがうっすら残り、軌跡（トレイル）が見え**ます。

| 透明度 | 効果 |
|--------|------|
| `255` | 完全にクリア — 残像なし |
| `40` | 半透明 — 軌跡が残る（このスケッチ） |
| `10` | かなり薄い — 長い軌跡 |

## フレームレートの変更

```js
p.setup = () => {
  p.createCanvas(400, 400);
  p.frameRate(30); // 30fps に変更
};
```

## 試してみよう

- `vx` と `vy` の値を大きくして速度を上げてみましょう
- `background()` の透明度を変えて軌跡の長さを調整してみましょう
- ボールのサイズをフレームごとに変化させてみましょう（例: `p.circle(x, y, 20 + p.sin(p.frameCount * 0.1) * 10)`）
